# 006: チャットUI(機能骨格)の実装

## 1. 目的
ブラウザからログインし、会話の作成・切り替え・削除と、
ストリーミング表示付きのチャット送受信ができる最小UIを実装する。

## 2. 対象
Phase 1「単一/複数セッションの作成・一覧・切り替え・削除」(UI側)
「llama.cpp serverへのストリーミング接続(トークン逐次表示)」(表示側)

## 3. 前提・参照
- 001〜005完了済み(認証API、会話CRUD API、SSEチャットエンドポイントすべて動作確認済み)
- 技術方針: ビルドツールなしの素のHTML/CSS/JS。フレームワーク・npmフロント依存は追加しない
- .base/public/js/api.js のJWT管理パターン(localStorage保存、Bearerヘッダー付与)は参考にしてよい。
  UI/レイアウトはPlainChat独自(ロールプレイ系UIは参照しない)
- SSE受信の注意: EventSourceはGET専用のため使えない。
  fetch + response.body.getReader() でSSEを自前パースする
  (event: delta / done / error、dataはJSON。バッファリングして行単位で処理し、
  チャンク境界がイベント途中で切れるケースに対応する)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### 構成(public/ 配下、既存index.htmlは置き換え)
- index.html : ログインビューとチャットビューを1ページに持つSPA風構成
- css/style.css : 最小限のスタイル
- js/api.js : トークン管理(localStorage)+ fetchラッパー(Bearer自動付与、401時はログインビューへ)
- js/app.js : 画面制御・会話一覧・チャット送受信

### ログインビュー
- username / password 入力 → POST /api/auth/login
- 成功でトークンをlocalStorageに保存しチャットビューへ
- 失敗時はエラーメッセージ表示
- ページロード時、トークンがあれば GET /api/auth/me で有効性確認し、
  有効ならログインをスキップ

### チャットビュー(左サイドバー+右チャットの2ペイン)
サイドバー:
- 「新しい会話」ボタン(POST /api/conversations → 一覧更新+その会話を開く)
- 会話一覧(updated_at降順)。クリックで切り替え、現在の会話をハイライト
- 各会話に削除ボタン(confirm付き、DELETE後は一覧更新。開いていた会話なら未選択状態へ)
- ログアウトボタン(トークン破棄→ログインビュー)

チャットペイン:
- 会話選択時、GET /:id/messages で履歴を表示(user右寄せ/assistant左寄せ程度の区別)
- 下部に入力欄(textarea)+送信ボタン。Enterで送信、Shift+Enterで改行
- 送信時: 自分の発話を即座に表示 → POST /:id/chat をfetchで開き、
  deltaを受信するたびにアシスタントの吹き出しへ逐次追記
- 受信中は送信ボタンを無効化。doneで通常状態に復帰
- event: error 受信時・fetch失敗時はエラー表示(アプリが固まらないこと)
- 新規メッセージで最下部へ自動スクロール
- 本文はtextContentで描画(innerHTMLに生文字列を入れない。XSS防止。
  Markdown描画は007で導入)

### 見た目の基準
- 007で磨くため最小限でよいが、2ペインレイアウトが崩れず日常操作に耐えること
- 配色はライトのみ(ダークモードは007)

## 5. やらないこと
- Markdown/コードブロックのレンダリング、シンタックスハイライト(007)
- ダークモード、デザインの磨き込み(007)
- 会話タイトルの編集UI・自動生成(タイトル変更APIは存在するがUIは007以降)
- 検索・フィルタ(Phase 2)
- 送信中の停止ボタン(007で検討)
- バックエンドの変更(既存APIで完結させる。不足を見つけたら実装せず報告)
- .env の変更

## 6. 完了条件
ブラウザ(http://localhost:18091)で以下を確認:
1. ログイン成功→チャットビュー、誤パスワード→エラー表示
2. リロードしてもログイン状態が維持される
3. 新しい会話を作成→メッセージ送信→アシスタント応答が逐次(タイプライター風に)表示される
4. 会話を2つ以上作り、切り替えでそれぞれの履歴が正しく表示される
5. 2往復目の送信で文脈の通じた応答が返る
6. 会話の削除が動作し、一覧から消える
7. ログアウト→ログインビューに戻り、リロードしてもチャットビューに入れない
8. <script>alert(1)</script> を含むメッセージを送信し、スクリプトが実行されず
   文字列として表示されることを確認(送信後、この確認用会話は削除)
9. 開発者ツールのコンソールに未処理エラーが出ていないこと
