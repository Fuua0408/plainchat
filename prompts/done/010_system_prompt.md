# 010: システムプロンプト設定(グローバル+会話ごと上書き)

## 1. 目的
全会話に適用されるグローバルのカスタムインストラクションと、
会話単位での上書き設定を実装し、応答の口調・振る舞いを制御可能にする。

## 2. 対象
Phase 2「システムプロンプト/カスタムインストラクションの設定」

## 3. 前提・参照
- 001〜009完了済み
- 適用規則: 会話ごと設定(conversations.system_prompt)が非NULLかつ非空なら
  それを使用 → なければグローバル(users.system_prompt) → どちらも空なら
  systemメッセージなし(現状どおり)
- タイトル生成(generate-title)にはシステムプロンプトを適用しない
- スキーマ変更はマイグレーションツールを使わず、db.jsの初期化SQLに
  カラムを含めつつ、既存DB向けに起動時の「カラム存在チェック+ALTER TABLE」で対応する
  (ユーザーの手作業に依存しない)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### スキーマ(src/db.js)
- users に system_prompt TEXT(NULL可)を追加
- conversations に system_prompt TEXT(NULL可)を追加
- CREATE TABLE文への追記+起動時にPRAGMA table_infoで両カラムの有無を確認し、
  なければALTER TABLE ... ADD COLUMNを実行(冪等であること)

### バックエンドAPI
- GET /api/settings → { system_prompt }(自分のusers.system_promptを返す。NULLは''で返す)
- PUT /api/settings → body: { system_prompt } でusers.system_promptを更新
  (空文字はNULLとして保存。長さ上限20000文字、超過は400)
- 会話ごと: 既存のPATCH /api/conversations/:id を拡張し、
  body: { system_prompt } も受け付ける(title/system_promptの片方だけでも可。
  空文字はNULL=グローバル継承に戻す。上限20000文字)
- GET /api/conversations/:id/messages のレスポンスに会話のsystem_promptを含める
  (例: { conversation: { id, title, system_prompt }, messages: [...] } の形に拡張。
  フロントの既存呼び出しへの影響に注意して両ファイルを整合させる)

### chat.js の変更(最小限)
- 履歴構築後、適用規則に従って messages 配列の先頭に
  { role: 'system', content: ... } を挿入(空なら挿入しない)
- DBに保存するのはuser/assistantメッセージのみ(systemは保存しない。
  設定変更が次の送信から反映されるようにするため)

### フロントエンド
- サイドバー下部(テーマ切替の近く)に「設定」ボタン → モーダルで
  グローバルのシステムプロンプトをtextareaで編集・保存(PUT /api/settings)
- チャットペイン上部に会話設定ボタン → モーダルで会話別システムプロンプトを編集
  (空のときはプレースホルダで「未設定(グローバル設定を使用)」と分かるように)
- モーダルは共通の簡素なもので可(オーバーレイ+閉じるボタン。Escで閉じる)
- 保存成功/失敗のフィードバックを表示(alertではなくモーダル内のメッセージ)

## 5. やらないこと
- プロンプトのテンプレート管理・プリセット機能
- generate-titleへのシステムプロンプト適用
- systemメッセージのDB保存・履歴表示
- 過去メッセージへの遡及的な影響(仕様として、変更は次の送信から反映)
- .env の変更

## 6. 完了条件
1. 既存DB(バックアップを取った上で)でサーバー起動 → ALTERが自動適用され、
   既存の会話・メッセージが無傷であること。再起動してもエラーが出ない(冪等)
2. グローバル設定に「語尾に必ず『ですわ』を付ける」等の検証しやすい指示を保存
   → 新規会話で応答に反映される
3. 会話別設定に別の指示を入れる → その会話のみ上書きされ、他の会話はグローバルのまま
4. 会話別設定を空にして保存 → グローバル継承に戻る
5. どちらも空の状態 → systemメッセージなしで従来どおり動作
   (LLMへの実リクエストにsystemが含まれないことを確認)
6. 設定変更がタイトル自動生成の挙動に影響しないこと
7. 20001文字の保存 → 400。トークンなしで各API → 401
8. 検証用のグローバル/会話別設定を空に戻し、テスト会話を削除して完了
