# 002: DB基盤とスキーマ構築

## 1. 目的
better-sqlite3によるDB初期化処理とPhase 1用スキーマを実装し、
起動時に初期ユーザーがシードされる状態を作る。

## 2. 対象
Phase 1「会話履歴のDB永続化」の土台。および認証(次タスク)の前提となるusersテーブル。

## 3. 前提・参照
- 001完了済み(Express起動、better-sqlite3はインストール済み)
- .base/src/db.js にNookResonanceのDB初期化パターンがある。
  接続管理(getDb)の構造は参考にしてよい。ただしスキーマは持ち込まない
  (character/affinity等のロールプレイ用テーブルは対象外)
- 設計方針: 単一ユーザー運用だが、全テーブルにuser_idスコープを持たせ、
  マルチユーザー対応可能な構造を保つ(DECISIONS.md 2026-07-05参照)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### src/db.js(新規)
- better-sqlite3で process.env.DB_PATH(デフォルト ./data/plainchat.db)に接続
- data/ ディレクトリがなければ作成
- PRAGMA journal_mode = WAL、PRAGMA foreign_keys = ON を設定
- getDb() で接続シングルトンを返す
- initDb() でスキーマ初期化+初期ユーザーシード(下記)

### スキーマ(CREATE TABLE IF NOT EXISTS)
users
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - username TEXT NOT NULL UNIQUE
  - password_hash TEXT NOT NULL
  - is_admin INTEGER NOT NULL DEFAULT 0
  - created_at TEXT NOT NULL DEFAULT (datetime('now'))
  - updated_at TEXT NOT NULL DEFAULT (datetime('now'))

conversations
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE
  - title TEXT NOT NULL DEFAULT '新しい会話'
  - created_at TEXT NOT NULL DEFAULT (datetime('now'))
  - updated_at TEXT NOT NULL DEFAULT (datetime('now'))
  - INDEX: (user_id, updated_at DESC)

messages
  - id INTEGER PRIMARY KEY AUTOINCREMENT
  - conversation_id INTEGER NOT NULL REFERENCES conversations(id) ON DELETE CASCADE
  - role TEXT NOT NULL CHECK(role IN ('system','user','assistant'))
  - content TEXT NOT NULL
  - created_at TEXT NOT NULL DEFAULT (datetime('now'))
  - INDEX: (conversation_id, id)

### 初期ユーザーシード(initDb内)
- usersテーブルが0件の場合のみ実行
- process.env.INITIAL_ADMIN_USER / INITIAL_ADMIN_PASSWORD から作成(is_admin=1)
- パスワードはbcryptでハッシュ(cost 12)
- 未設定の場合はエラーログを出してシードをスキップ(起動自体は継続)
- シード成功時・スキップ時ともにloggerで明示的にログを出す

### src/index.js の変更
- 起動時に initDb() を呼ぶ(listenの前)

### .env.example への追記
INITIAL_ADMIN_USER=admin
INITIAL_ADMIN_PASSWORD=change-me
