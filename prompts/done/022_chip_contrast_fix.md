# 022: ユーザーバブル内ファイルチップの視認性修正

## 1. 目的
ユーザー発言バブル内のファイルチップが「白地に白文字」で読みにくい問題を修正する。
チップの配色をバブルの文字色継承から切り離し、両テーマで確実に読めるようにする、
CSSに閉じた小さな修正。

## 2. 対象
Phase 3 ②「ファイル添付」の磨き込み。018起点の既存CSS問題(020の実行報告で指摘)。

## 3. 前提・参照
- 013〜021完了済み。ファイルチップは018で実装(public/css/style.css)。
  018/020の報告によると、ユーザーバブル内チップは背景が var(--bg-surface) である一方、
  文字色をユーザーバブル(青背景用の白文字)から継承しており、ライトテーマで
  白地に白文字になり読めない
- チップは「送信前プレビュー」「送信直後のユーザーバブル内」「履歴のユーザーバブル内」で
  表示される。問題が出るのはユーザーバブル(青背景)内のチップ
- 技術方針: ビルドなしの素のHTML/CSS/JS。CSS変数(テーマトークン)を使う
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件
- ユーザーバブル内のファイルチップについて、文字色をバブルからの継承(白文字)に
  依存させず、チップ自身の背景色に対して読めるコントラストを持つ配色に固定する
  (チップの背景・文字色・必要なら枠線を、チップ側で明示する)
- ライトテーマ・ダークテーマの両方で、ユーザーバブル内チップの文字・アイコン・
  サイズ表記がすべて判読できること
- 修正は配色(background / color / border 程度)に限定する。チップの形状・余白・
  レイアウト・アイコンなどのデザインは変更しない

## 5. やらないこと
- チップのデザイン刷新(形・余白・アイコン・角丸等の変更)
- 画像サムネイル表示・履歴描画・チップ生成ロジック(JS)の変更
- assistant側バブルや、プレビュー領域のチップが既に読める場合、その配色の変更
  (問題箇所=ユーザーバブル内チップに限定。他が壊れないことを優先)
- テーマトークン(CSS変数)の定義そのものの変更(既存変数を使う。新規追加は最小限)
- バックエンド・.env の変更

## 6. 完了条件
ブラウザ(http://localhost:18091)で確認:
1. ライトテーマで、ファイルを添付送信 → ユーザーバブル内チップのファイル名・サイズが
   はっきり読める(白地に白文字が解消)
2. ダークテーマでも同チップが読める
3. リロード後の履歴のユーザーバブル内チップも、両テーマで読める
4. 送信前プレビューのチップ、画像サムネイル、assistant側の表示が従来どおり
   崩れていない(リグレッションなし)
5. コンソールに未処理エラーがない
6. テストで作成した会話・添付を削除して完了
