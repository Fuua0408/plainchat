# 020: 履歴ファイルチップの実名表示(フロント追従)

## 1. 目的
履歴のファイルチップに、019で返るようになった original_name(実ファイル名)を表示する。
019のバックエンド追補を画面へ反映する、小さなフロント追従。これでPhase 3 ②の仕上げが完結する。

## 2. 対象
Phase 3 ②「ファイル添付」の仕上げ。019(messages attachmentに original_name/size 追加)への追従。

## 3. 前提・参照
- 013〜019完了済み。GET /api/conversations/:id/messages の各 attachment は
  { id, kind, mime, url, original_name, size } を返す(019)。original_name は NULL の場合あり
- 018でファイルチップの表示・履歴描画・本文プレビューは実装済み。現状、履歴側チップは
  original_name が無かったため mime からの種別ラベルにフォールバックしている
- public/js/app.js の履歴描画(renderMessages / appendMessageBubble 系)と、
  ファイルチップ生成箇所が対象
- 技術方針: ビルドなしの素のHTML/CSS/JS。ファイル名は必ず textContent で描画(innerHTML禁止)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件
- 履歴のファイルチップの表示名を、original_name があればそれを使う。
  original_name が null/空のときのみ、現行の mime 由来の種別ラベルにフォールバックする
- size が返る場合、チップにサイズを併記してよい(任意。送信直後の表示と体裁を揃える程度)
- 送信直後(手元にFileがある)の表示と、履歴再描画時の表示で、ファイル名の見え方が
  一致すること(どちらも実名が出る)
- ファイル名は textContent で描画し、XSS耐性(018の方針)を維持する

## 5. やらないこと
- バックエンドの変更(019で完結)
- チップのデザイン刷新、本文プレビュー機能の変更(表示名の解決だけ直す)
- 画像側の表示ロジックの変更
- .env の変更

## 6. 完了条件
ブラウザ(http://localhost:18091)で確認:
1. ファイルを添付送信 → 送信直後もリロード後も、チップに実ファイル名が表示される
2. 会話を切り替えて戻っても実名が維持される(種別ラベルに戻らない)
3. original_name が null の添付(直接DB作成等)では、種別ラベルにフォールバックし
   エラーにならない
4. ファイル名に <img onerror=...> 等を含めてもスクリプトが実行されない(textContent描画)
5. コンソールに未処理エラーがない
6. テストで作成した会話・添付を削除して完了

- git add / git commit は実行しない(ステージもしない)。コミットはレビュー後にユーザーが手動で行う
- 実装と動作確認が完了したら、この指示ファイルを prompts/done/ へ移動する
