# 007: UI磨き込み(Phase 1完成)

## 1. 目的
Markdown/コードブロック描画・ダークモード・UX改善を実装し、
Phase 1のチェックリストを完成させる。

## 2. 対象
Phase 1「Markdown/コードブロックのレンダリング」+006で発見されたUX課題

## 3. 前提・参照
- 001〜006完了済み(チャットUI骨格が動作中)
- 技術方針: ビルドなしの素のHTML/CSS/JS。ライブラリはCDN読み込みのみ
  (marked / highlight.js / DOMPurify を想定。バージョン固定のURLを使う)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### A. Markdownレンダリング(アシスタントメッセージのみ)
- markedでHTML化 → **必ずDOMPurifyでサニタイズしてから**innerHTMLに設定
- ユーザーメッセージは従来どおりtextContent(プレーン表示)
- コードブロックはhighlight.jsでハイライト+右上にコピーボタン
- ストリーミング中の逐次描画: delta受信ごとに蓄積テキストを再レンダリング
  (簡易で可。ちらつきが過度なら受信中はプレーン表示・done時にMarkdown化でも可、
  採用した方式を報告すること)
- 表示中の既存履歴にも適用される(過去メッセージの再表示時)

### B. 応答待ちインジケータ(006課題3)
- 送信後、最初のdeltaが届くまでアシスタント側に「考えています…」
  (アニメーション付きドット等)を表示
- 最初のdelta受信で本文表示に切り替え

### C. 未選択時の送信で自動新規作成(006課題1)
- 会話未選択状態で送信したら、エラーではなく自動で会話を作成してから送信する
  (POST /api/conversations → 成功後にその会話へ送信、一覧も更新)

### D. ダークモード
- ライト/ダークの切り替えトグル(サイドバー下部)
- 初期値は prefers-color-scheme に従い、選択はlocalStorageに保存
- CSS変数(カスタムプロパティ)でテーマ管理し、両テーマで全要素が読めること
  (コードブロックのハイライトテーマも切り替えに追従)

### E. 会話タイトルの手動編集
- 会話一覧の項目にリネーム操作(ダブルクリックまたは編集ボタン)
  → PATCH /api/conversations/:id
- 空文字は不可(既存APIのバリデーションに従いエラー表示)

### F. 送信中の停止ボタン
- ストリーミング受信中は送信ボタンを「停止」に切り替え
- 停止でfetchをAbortControllerで中断(バックエンドは部分応答を保存する仕様のため、
  停止後にメッセージ再取得で部分応答が表示されることを確認)

## 5. やらないこと
- バックエンドの変更(既存APIで完結。不足があれば実装せず報告)
- 会話検索・フィルタ、タイトル自動生成(Phase 2)
- システムプロンプト設定(Phase 2)
- npmへのフロント依存追加(CDNのみ)
- .env の変更

## 6. 完了条件
ブラウザで確認:
1. Markdown(見出し・リスト・太字・リンク)が整形表示され、
   コードブロックがハイライト+コピーボタン付きで表示される
2. <script>タグや onerror属性を含む入力を送ってもスクリプトが実行されない
   (DOMPurify経由の確認。ユーザー発話・アシスタント応答の両経路)
3. 送信直後に「考えています…」が出て、最初のトークンで本文に切り替わる
4. 会話未選択で送信 → 自動で新規会話が作られ、正常に応答が返る
5. ダーク/ライト切り替えが全画面に効き、リロード後も維持される
6. タイトルのリネームが動作し、一覧に反映される
7. ストリーミング中に停止 → 中断され、部分応答が履歴に残る
8. コンソールに未処理エラーがない
9. テストで作成した会話は削除して完了

## 7. 共通ルール
- git add / git commit は実行しない(ステージもしない)。コミットはレビュー後にユーザーが手動で行う
- 実装と動作確認が完了したら、この指示ファイルを prompts/done/ へ移動する
