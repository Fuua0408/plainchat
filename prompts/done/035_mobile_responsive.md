# 035: モバイルレスポンシブ対応(最小・破綻解消)

## 1. 目的
スマホ幅(狭い画面)で PlainChat のUIが破綻する問題を解消する。PC前提の固定2カラム(左:会話リスト/
右:チャット)を、狭い幅ではサイドバーをドロワー化+チャットを全幅にし、入力欄・ボタンの潰れを直す。
「普通に使える」ラインまでの最小対応。PC の見た目・挙動は一切変えない。

## 2. 対象
v1後の小改修(035)。フロント(public/)のみ。chat.js・src/mcp/*・DB・認証・API には触れない。
Android(Pixel等)ブラウザ/PWA standalone での実用が目的。

## 3. 前提・参照
- public/ は素の HTML/CSS/JS(ビルドなし)。index.html + css/style.css + js/app.js。marked/DOMPurify/
  highlight.js は CDN
- 現状は PC 前提の固定2カラムレイアウト。スマホ幅ではサイドバーが画面を占有しチャットがはみ出す、
  入力欄「メッセージを入力」が縦に潰れて1文字ずつ折り返す、送信/添付ボタンのレイアウトが崩れる
- UIテーマ色 #2563eb。ライト/ダーク両モードあり(「ダーク」トグル)。034 で PWA standalone 対応済み
- 既存要素: サイドバー(新しい会話/会話検索/日付絞り込み/会話一覧/ダーク/設定/MCPサーバー設定/ログアウト)、
  会話設定ボタン、チャット表示領域、入力欄(添付・ファイル・メッセージ入力・送信)、各種モーダル
- prompts/ 配下は指示書。移動以外の編集はしない

## 4. 要件

### (a) ブレークポイントで切り替え(PC無干渉)
- CSS メディアクエリで「狭い幅(例: max-width 768px)」のときだけモバイルレイアウトを適用する
- 広い幅(PC)では現行の2カラムレイアウト・見た目・挙動を完全に維持(既存スタイルを変更せず、
  メディアクエリ内で上書きする方針。PC 表示のリグレッションを出さない)

### (b) サイドバーのドロワー化(狭い幅のみ)
- 狭い幅では、サイドバーを既定で隠し、チャット領域を全幅表示にする
- ハンバーガーボタン(☰)をヘッダ等の押しやすい位置に追加し、タップでサイドバーを左からドロワー表示
  (オーバーレイで背面を薄暗くし、オーバーレイタップまたは閉じるボタンで閉じる)
- 会話を選択したら、または「新しい会話」を押したら、ドロワーを自動で閉じてチャットに戻る
- ドロワー開閉のトグルは js/app.js に最小限のロジックを追加(クラス付け外し等)。既存のイベント配線を壊さない
- ハンバーガーボタンは PC 幅では非表示(サイドバー常時表示のため不要)

### (c) 入力欄・ボタンの潰れ解消(狭い幅のみ)
- 入力欄「メッセージを入力」が縦に潰れて1文字ずつ折り返す現象を解消(横幅を確保し、複数行入力が
  自然に扱えるようにする)
- 添付・ファイル・送信ボタンが横並びで潰れずに収まるよう再配置(必要ならアイコンサイズ/余白を調整)。
  送信ボタンが画面外に切れないこと

### (d) はみ出し・横スクロールの解消
- 狭い幅で横方向のはみ出し/意図しない横スクロールが起きないこと(2カラムの固定幅が原因のはみ出しを解消)
- 「新しい会話」ボタン等が画面端で切れないこと

### (e) ライト/ダーク両対応
- モバイルレイアウトがライト/ダーク両モードで崩れず、コントラスト・可読性が保たれること
- ドロワーのオーバーレイ・背景色も両モードで破綻しないこと

## 5. やらないこと
- chat.js・src/mcp/*・DB・認証・API・LLM 呼び出しの変更
- PC(広い幅)レイアウトの見た目・挙動の変更(モバイル用メディアクエリ内でのみ変更する)
- タップ領域の細かな最適化・フォント/余白の全面的な作り込み・モーダルのモバイル最適化などの快適性向上
  (今回は「破綻解消」まで。快適性は実機で触ってから別途)
- Service Worker、オフライン対応
- 機能追加(会話・ツール・MCP 等の挙動は不変)

## 6. 完了条件
（検証は PC 幅とモバイル幅の両方で行う。ブラウザの開発者ツールのデバイスエミュレーション or 実機)
1. PC 幅(広い)で、現行と見た目・挙動が同一(2カラム・サイドバー常時表示・入力欄・モーダル等)。リグレッションなし
2. モバイル幅(例 375〜412px 相当)で、初期表示はチャット領域が全幅、サイドバーは隠れ、横スクロール/
   はみ出しが無い
3. ハンバーガーでサイドバーがドロワー表示され、オーバーレイタップ/閉じるで閉じる。会話選択・新しい会話で
   自動的に閉じてチャットに戻る
4. 入力欄が縦潰れ(1文字折り返し)せず、メッセージ入力・添付・ファイル・送信ボタンが潰れず画面内に収まり操作できる
5. ライト/ダーク両モードで 2〜4 が崩れない
6. 既存機能(会話作成/選択/検索/日付絞り込み/送信/添付/MCP設定/ダーク切替/ログアウト)がモバイルでも
   従来どおり動作する(レイアウト変更で操作不能になった要素がない)
7. 【ユーザー手動】Pixel 等の実機(ブラウザ+PWA standalone)で開き、破綻せず普通に使えることを確認
   (この実機確認はユーザーが行う。Claude Code 側はエミュレーション/幅指定で 1〜6 を確認)

## 7. 共通ルール
- git add / git commit は実行しない(ステージもしない)。コミットはレビュー後にユーザーが手動で行う
- 実装と動作確認が完了したら、この指示ファイルを prompts/done/ へ移動する
