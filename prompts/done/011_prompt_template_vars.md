# 011: システムプロンプトのテンプレート変数展開

## 1. 目的
システムプロンプト内の {{currentDateTime}} 等のテンプレート変数を、
送信時に実際の値へ展開してLLMに渡す。

## 2. 対象
Phase 2(010 システムプロンプト設定の拡張)

## 3. 前提・参照
- 010完了済み。src/routes/chat.js がsystemメッセージを動的挿入している
- 展開は「LLMへ送る直前」に行う。DBに保存された設定値は変数のまま保持する
  (設定画面では変数表記が見え、送信ごとに最新の値が入る)
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### 変数展開(src/ 配下に小さなユーティリティとして実装)
サポートする変数(大文字小文字は区別する):
- {{currentDateTime}} → 例: 2026-07-05 21:30 (土曜日)
- {{currentDate}}     → 例: 2026-07-05 (土曜日)
- {{currentTime}}     → 例: 21:30
サーバーのローカルタイムゾーンを使用。未知の {{...}} は展開せずそのまま残す。

### 適用箇所(src/routes/chat.js)
- systemメッセージ挿入時(グローバル/会話別のどちらの場合も)、
  contentに変数展開を適用する
- user/assistantメッセージには適用しない(ユーザー本文の {{...}} を書き換えない)

## 5. やらないこと
- generate-titleへの適用(システムプロンプト自体を使っていないため)
- 変数の追加設定UI、ヘルプ表示(設定モーダルへの説明追加は任意で可)
- WebSearch等の新機能(将来の別タスク)
- .env の変更

## 6. 完了条件
1. グローバル設定に「現在日時は {{currentDateTime}} です。日時を聞かれたらこれに基づいて
   答える」を保存 → チャットで「今日は何日?」と聞くと実際の日付を答える
2. モックまたはリクエストキャプチャで、LLMへ送られたsystemメッセージ内の変数が
   展開済みであることを直接確認
3. DB上の設定値は {{currentDateTime}} のまま(設定モーダルを開いても変数表記のまま)
4. ユーザー発話に {{currentDateTime}} と書いて送信 → 展開されずそのまま扱われる
5. 未知変数 {{foo}} を含む設定 → エラーにならず {{foo}} のまま送られる
6. 検証用設定を元に戻し、テスト会話を削除して完了

## 7. 共通ルール
- git add / git commit は実行しない(ステージもしない)。コミットはレビュー後にユーザーが手動で行う
- 実装と動作確認が完了したら、この指示ファイルを prompts/done/ へ移動する
