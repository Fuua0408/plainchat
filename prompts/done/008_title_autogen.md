# 008: 会話タイトルの自動生成

## 1. 目的
最初の往復が完了した会話に、内容を要約した短いタイトルをLLMで自動生成して付与する。
サイドバーに「新しい会話」が並ぶ状態を解消する。

## 2. 対象
Phase 2「会話タイトルの自動生成」

## 3. 前提・参照
- Phase 1完了済み。src/routes/chat.js のLLM接続パターン
  (env読み込み、Connection: close、エラーハンドリング)を参照
- 007検証時の知見: 使用モデルは隠れ思考(reasoning_content)にトークンを使い切り
  contentが空になることがある。小さいmax_tokensでは致命的なため、
  タイトル生成リクエストには enable_thinking: false を必ず含める
- 設計方針: 既存の chat.js には手を入れない。タイトル生成は独立エンドポイントとし、
  失敗してもチャット本体の体験に影響させない
- prompts/ 配下は指示書であり、移動以外の編集はしない

## 4. 要件

### バックエンド: POST /api/conversations/:id/generate-title
(src/routes/conversations.js に追加、authMiddleware+所有者チェックは既存と同方針)

処理:
1. 所有者チェック(なければ404)
2. 会話の最初のuserメッセージと最初のassistantメッセージを取得
   (assistantがまだ無い場合は400 { error: 'No messages to summarize' })
3. 各メッセージは先頭500文字に切り詰めてLLMへ渡す
4. LLMへ非ストリーミングでリクエスト:
   - システム指示: 「以下の会話に短いタイトルを付ける。会話と同じ言語で、
     10〜20文字程度、記号や引用符や説明は不要、タイトルのみを出力」の趣旨
   - max_tokens: 60、temperature: 0.3、enable_thinking: false
   - タイムアウト30秒、Connection: close(chat.jsと同じ理由)
5. 応答を整形: 前後の空白・引用符・改行を除去、先頭行のみ採用、50文字で切り詰め
6. 空になった場合やLLMエラー時は502 { error: ... }(タイトルは変更しない)
7. 成功時: conversations.title を更新(updated_atは変更しない。
   並び順はメッセージ送受信の時刻を反映すべきで、タイトル付与で動かさない)し、
   { conversation: { id, title, ... } } を返す

### フロントエンド(public/js/app.js, api.js)
- チャットの done 受信後(onDone内)、現在の会話のタイトルが
  デフォルト(「新しい会話」)の場合のみ generate-title を呼ぶ
- 成功したらサイドバーの一覧を更新
- 失敗(エラーレスポンス/ネットワークエラー)は console.warn に留め、
  ユーザーにはエラー表示しない(デフォルトタイトルのまま)
- 生成中のスピナー等は不要(バックグラウンド処理として扱う)

## 5. やらないこと
- chat.js(SSEエンドポイント)の変更
- 2往復目以降でのタイトル再生成、手動での再生成ボタン(将来検討)
- タイトル生成用の別モデル・別エンドポイント設定(既存のLLM_*設定を使う)
- 会話の検索・フィルタ(Phase 2の別タスク)
- .env の変更

## 6. 完了条件
1. 新規会話で1往復 → 数秒以内にサイドバーのタイトルが内容を反映したものに変わる
   (curlでの確認でも可: chat完了後にgenerate-titleを叩き、titleが更新される)
2. 既にタイトルが手動変更された会話では、フロントがgenerate-titleを呼ばない
   (コード上の分岐確認+実際の操作で確認)
3. assistantメッセージが無い会話へのgenerate-title → 400
4. 他人/存在しない会話 → 404、トークンなし → 401
5. LLM_ENDPOINTを無効値にした状態でgenerate-title → 502が返り、
   タイトルが変わらないこと → .env復元(この確認のための一時変更は許可)
6. 生成されたタイトルに引用符・改行が含まれないこと(日本語/英語の両方で1回ずつ確認)
7. テストで作成した会話は削除して完了
